0. Introduction

This report will focus on the e-commerce website for JD Sports, jdsports.ie. The report will analyze its usability/accessibility based on the 12 guidelines layed out in the Web Content Accessibility Guidelines, grouped under four general principles:
Perceivable, Operable, Understandable, and Robust.

1. Perceivable

1.1. Text Alternatives

Inspection of the source code for the site shows that every image on the website has an ALT tag. This makes the site more accessible to those with visual disabilities, as it allows the images on the site to be read out by a screen reader.

1.2. Time-Based Media Alternatives

On the frontpage of the website, there is a video that autoplays. Unfortunately, there is no alternative way to obtain the information shown in this video. This is poor accessibility, as it prevents those cannot hear or see the video from obtaining the information displayed in this video.

1.3. Multi-Presentable Content

The content of this website could be broken down into simple text and it would be possible to still use and understand the information displayed on the site. While the orientation, size, and shape of objects on the site are helpful, they are not necessary for understanding how to use the site.

1.4. Seperate Forground and Background

Colour is frequently used as the only visual cue to indicate an action being carried out (e.g. adding an item to the basket using 'Quick Buy', pressing the 'Back to Top' button.) This makes the site less accessible to those with colour-blindness. Also, on the front page of the website, there is a video that autoplays as soon as the page is loaded. There is no way to stop this video from playing without right-clicking it, and it will continue to loop until you stop it. There is also no way to lower the volume of the video independent of the overall system volume level. This is incredibly poor accessibility, making it quite difficult for a blind person to navigate the site as this video may overpower the sound of their screen reader.

2. Operable

2.1. Keyboard Functionality

It is possible to use this website with only input from the keyboard, though there is little-to-no visual indication of what is being selected at a given time. This is a minor issue as most people who would use the keyboard-only are those with visual disabilities.

2.2. Time to Interact with Content

For the majority of the website, there are no time limits set on content. However, the frontpage contains a video that autoplays. There is no way to pause this video without right-clicking and selecting an option to show controls. This is incredibly inconvenient, especially to those with visual disabilities.

2.3. Seizure-Inducing Content

No flashing images, videos, or anything known to cause seizures is found on the site.

2.4. Navigation

Navigation around the site is simple to understand. There is a header at the start of every webpage with several different headings (e.g. 'Men', 'Women', etc...). Hovering over any of these headings causes a drop-down list to appear, in which there are dozens of links to other webpages grouped under more headings (e.g. 'Footwear', 'Clothing', etc...). Each of these webpages also have titles that clearly describe what is contained on them.

3. Understandable

3.1. Readable and Understandable Text

Each webpage contains a lang attribute in the HTML tag that specifies the language of the webpage. There are no other areas in the webpage that changes language from the one specified in the HTML tag.

3.2. Predictable Web Pages

Most the webpages on this website act in predictable ways. There is no situation on any of the pages that changes context with a change in focus. However, when adding an item to the basket, the website automatically moves the user to the basket webpage. There is no indication that when the user clicks 'Add to Basket' that they will be forced to go to the basket webpage. If, on the other hand, the user uses the 'Quick Buy' button to add an item to the basket, they are not automatically transported to the basket webpage. This is poor usability, as it can easily confuse a user, and is often very irritating in the former case if the user wants to continue browsing the website.

3.3. Avoid and Correct Mistakes

If the user uses the search bar to search for an item that does not exist on the site, they are transported to an error page that tells them the product specified could not be found. It is also not possible to enter a wrong value in the basket webpage, as when choosing the quantity the user can only press the + or - buttons, they are not allowed to enter a value themselves. All content requiring user input is also clearly labeled.

4. Robust

4.1. Compatibility

All elements on the website have complete start and end tags as per compatibility standards. All IDs are unique, all elements are nested as needed, and there are no duplicate attributes. It also possible to programmatically obtain the name and role of user interface components, and programmatically set the states, properties, and values of these components.

5. Conclusion

To conclude, jdsports.ie passes all Priority 1 checkpoints, meaning it is possible for all groups of users to access. However, it fails all Priority 2 checkpoints, meaning it will be very difficult for several groups of users to access. For the e-commerce site of a popular sports store, this is very disappointing. To fail some Priority 2 checkpoints could be overlooked, button failing all of them is unacceptable. It should not be difficult for such a well-established brand to create a site that passes all of these checkpoints.
